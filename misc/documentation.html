Welcome to the documentation page of the PSite Suite ver. 2.03 (28.04.2004)

The PSite Suite is a tool for webmasters and server-administrators, it replaces the default indexing engine and shows
the directory-content split into areas:
  • The First area is the navigation. Here all available directorys will be listed.
  • Next there is the image-area, where all images contained in the directory are shown as thumbnails.
    You can choose to automatically generate thumbnails in a subdirectory or to display the full-sized
    pictures shrinked to thumbnail-size. If thumbnail-generation is turned on, the script will check if the
    full-sized picture have been changed or the thumb-sizes in the config have been altered, and if,
    it will regenerate the thumbnail.
  • The last area finally lists the non-image files in the directory along with their filesizes.
You can supply a readme file that's content will be displayed on one of four positions of the page you can choose, you
can configure each directory with different settings, like the colors, wich fonts to use in wich size and so on.

The Config File:
The config-file is called '.pss.conf' and should be in the root of your web and in every directory you want to configure
in a different way than the rest of you web. All values missing in one of these config-files will be replaced by the default
values of the webserver. Please note, that indexing is turned of by default. The config-file is divided into serveral sections
like [global], [body] or [table]. You don't need to have all keys and/or sections in your file, missing values are taken from
the default-config-files.

Let me explain the keys and values of the config-file showing them with the default values:

Section [global]:
  • showhidden = false
    You can 'hide' a file/dir from the listing if the filename/dirname starts with a period ( . ),
    if this is set to true, files/dirs starting with a period WILL be listed.
  • showindex = false
    Disables the indexing by default. If you set this to true, listing will be enabled.
  • readme_name = ".readme"
    Filename of the readme-file to be displayed by default.
  • show_navigation = true
    If set to false, the navigation won't be displayed
  • show_readme = true
    If set to false, the readme (if exist) won't be displayed
  • show_thumbs = true
    If set to false, the thumbs (if exist) won't be displayed
  • show_files = true
    If set to false, the files (if exist) won't be displayed
  • show_search = true
    If set to false, the search-form (if the search is enabled) won't be displayed
  • item_1 = "navigation"
    First section to display.
  • item_2 = "readme"
    Second section to display.
  • item_3 = "thumbs"
    Third section to display.
  • item_4 = "files"
    Fourth section to display.
  • rotate = false
    If set to true the script will also generate rotated versions of your images.
  • nav_sort = "asc"
    asc / desc
Section [table]:
  • limitrows = true
  • position_pagesnav = "both"
    top / bottom / both
  • maxcols = 5
    How many cols should the images-table have?
  • maxrows = 5
Section [imgview]:
  • limitsize = false
    Set to true if you want to limit the size in wich pictures are displayed
  • maxwidth = 800
    Maximum width of pictures, when limited (numeric)
  • maxheight = 600
    Maximum height of pictures, when limited (numeric)
  • allow_osize = false
    Allow viewing limited pictures in original size
  • show_navi = true
    Show image-navigation if in view-mode
  • show_name = true
    Show filename in view-mode
  • show_dimensions = true
    Show image-dimensions in view-mode
  • show_type = true
    Show image-type in view-mode
  • show_size = true
    Show filesize in view-mode
  • show_date = true
    Show filedate in view-mode
  • show_link = true
    Show link in view-mode
  • show_oimage = true
    Show original image-data when displaying a resized image
  • comment_position = bottom
    Where to display the image-comment (only if comments enabled) bottom / right
  • comment_width = 150
    How much space should be reserved for image-comment (only if comments enabled and displayed on the right)
  • position_navi = both
    Where should the navigation be displayed (both / top / bottom)
  • click_links_back = true
    Click on the image sends you back to the index (closes popup)
Section [thumbs]:
  • maxwidth = 150
    Maximum width for the thumbnails.
  • maxheight = 150
    Maximum height for the thumbnails.
  • show_name = true
    Show filename in thumbs-listing
  • show_date = true
    Show filedate in thumbs-listing
  • show_dimensions = true
    Show image-dimensions in thumbs-listing
  • show_size = true
    Show filesize in thumbs-listing
  • link_name = true
    Link the filename directly to the image (no further processing, though the image itself will still be linked
    to view-mode)
  • show_help = true
    Show helptext in thumbs-listing
  • show_number = true
    Show number of images in thumbs-listing
  • show_generated = true
    If new thumbs have been generated, display the number at the bottom of the thumbs-listing
  • crop_extension = false
Section [popup]:
  • enable = true
    Show images in popup-windows
  • width = 810
    Width of popup-window
  • height = 685
    Height of popup-window
  • auto_resize = true
  • auto_resize_width = 25
  • auto_resize_height = 135
Section [directory]:
The directive here is meant ONLY für subdirectories, you won't break anything if you put it somewhere
else, but also, nothing will happen by this. This is meant for sites that want to use the script in
subdirectories only. Set it to false in the directory in wich you dont want to link to the parent dir.
  • show_parent_link = true
    Enable/disable linking to the parent directory.

The .readme file:
You can put any text or html into this file, excluding the complete head and the body-tags. The content of this file
will NOT be parsed by any script-engine.

CSS Keys:
.pss_th_na_le: thumbs navi left
.pss_th_na_num: thumbs navi numbers
.pss_th_na_re: thumbs navi right
